# 项目简述

## 1. 背景

- 业务背景：制造业采购场景里，采购合同、供应商报价单和 PO 常由不同角色、系统和模板生成，人工逐项核对成本高且容易漏掉主体、付款条件、税率、交期等关键异常。
- 为什么现在要做：项目要作为私有化部署的业务型 AI Agent 样板，证明本地部署、Agent 工作流、RAG、规则校验、人工复核和留痕能力可以组合成完整闭环。
- 当前痛点或机会：现有审查过程缺少统一初审标准，人工复核结果和最终结论也缺少结构化留痕；这类问题适合做成可解释的规则型 Agent，而不是聊天式问答产品。

## 2. 目标

- 主要目标：支持 `采购合同`、`供应商报价单`、`PO` 三类文档的统一审查，完成字段抽取、跨文档一致性校验、风险识别、人工复核和报告输出。
- 成功标准：
  - 固定支持 8 类正式风险识别和 2 类非风险型人工复核场景。
  - 在 20 组标准样本上覆盖 `PASS`、`REJECT`、`MANUAL_REVIEW` 三种主结论。
  - 清楚展示 `AI 初审 + 人工复核 + 留痕` 的端到端业务流程。
- 目标用户或相关方：
  - 采购专员：上传文档、查看初审结果、修正单据。
  - 采购经理：复核高风险或多风险异常。
  - 法务：复核关键条款缺失和合同完整性问题。
  - 财务/内控：复核付款条件、币种、税率等结算类问题。
  - 产品/工程/实施团队：将该项目作为样板与可复用方案。

## 3. 范围

### 范围内

- 固定接收 `contract`、`quotation`、`po` 三类文档组成的 `DocumentPackage`。
- 抽取 PRD 定义的统一对象：`ExtractedDocument`、`LineItem`、`ReviewFinding`、`ReviewReport`。
- 固定对供应商、采购方、币种、税率、付款条件、交期、收货地点和行项目关键字段执行跨文档比对。
- 固定支持 8 类正式风险规则：`R001-R008`。
- 固定支持 2 类人工复核规则：`MR001-MR002`。
- 输出面向采购审查流程的结构化报告，并保留 AI 初审、人工复核和最终结论的留痕。

### 范围外

- 不处理合同谈判建议生成。
- 不做自动审批、自动下单或写回审批流。
- 不接入 ERP、OA、SRM 的正式写回流程。
- 不扩展到发票、收货单、入库单等第四类及以上单据。
- 不在 V1 中展开复杂 OCR 版面还原、印章真伪识别和聊天式主交互。
- 不新增第 9 类正式业务风险。

## 4. 系统边界

- 上游系统或输入：
  - 用户上传的采购合同、供应商报价单、PO。
  - 项目内置的 20 组标准样本，分为 `single_risk`、`multi_risk`、`clean`、`borderline` 四类。
- 下游系统或输出：
  - `ReviewReport` 结构化审查结果。
  - 人工复核记录与最终结论留痕。
  - 面向采购流程的报告展示结果。
- 明确边界或非目标：
  - V1 以私有化部署样板项目为目标，不承诺真实生产环境集成。
  - RAG 仅用于规则依据、条款模板和自然语言条款标准化辅助，不替代正式规则引擎。
  - 人工复核是系统流程的一部分，不可被删除为纯自动化流程。

## 5. 约束

- 业务约束：
  - 主结论固定为 `PASS / REJECT / MANUAL_REVIEW` 三态。
  - `REJECT` 由正式风险触发，原则上不能在同一轮审查中直接改为 `PASS`。
  - 边界样本只用于扩展人工复核机制，不扩展正式风险范围。
- 技术约束：
  - 行项目匹配优先使用 `item_code`，退化到 `item_name + spec_model`。
  - 无法唯一匹配时不能自动判定为正式风险，必须进入人工确认。
  - 近似名、自然语言交期或付款条款语义对齐等情况，只能作为人工确认场景处理。
- 交付或合规约束：
  - 报告与界面默认面向中文业务场景。
  - 审查流程必须可解释、可复核、可留痕。
  - 当前项目交付阶段仍是方案定义，尚未形成可运行系统。

## 6. 当前状态摘要

- 已经存在的内容：
  - V1 PRD 已定义业务目标、风险范围、对象模型、样本边界和验收标准。
  - 项目内已存在 20 组合同/报价单/PO 样本组。
  - 核心文档骨架已经建立。
- 仍然缺失的内容：
  - 系统实现、规则引擎、报告渲染、人工复核界面和留痕机制尚未落地。
  - 20 组样本的实际跑通验证尚未记录。
  - 与外部系统集成、部署方式和接口细节仍未展开。
- 最近的下一里程碑：基于已对齐文档，进入 V1 系统设计与实现拆解，优先明确对象模型落地、规则执行链路和报告输出方案。
